# 测试 Artifact 契约

本契约用于约束 `测试方案.md`、`测试用例评审.md`、`执行记录.md` 和 `测试报告.md` 的可追踪关系。reporter 生成最终结论前，必须优先使用仓库根目录的 `scripts/validate_all.py` 检查本契约；`skills/api-blackbox-tester/scripts/validate_test_artifacts.py` 和 `skills/api-blackbox-tester/scripts/check_report_gate.py` 仅作为兼容入口保留。

如果报告结论不是 `通过`，必须进入非通过闭环；报告需要给出根因分析、解决建议、责任阶段、复验范围和下一次 executor 复验输入。

## 必须存在的阶段文件

同一测试目录中应包含：

- `测试方案.md`
- `测试用例评审.md`
- `执行记录.md`
- `测试报告.md`

缺少前序阶段文件时，报告不得结论为 `通过`。

## ID 前缀建议

- 业务场景：`SCN-001`
- 场景覆盖单元：`SCENE-001`
- 普通用例：`API-001`
- 边界用例：`BND-001`
- 安全用例：`SEC-001`
- 矩阵单元：`MATRIX-001`
- 跨接口一致性：`CONS-001`
- 编排链路：`FLOW-001`
- DB 验证：`DB-001`
- 问题：`ISSUE-001`
- 证据包：`EVI-001`
- 断言：`ASSERT-001`
- 影响项：`IMP-001`

## Planner 契约

`测试方案.md` 必须包含：

- `结构化路由影响分析`
- `业务场景拆解`
- `变更代码影响接口清单`
- `影响范围初判`
- `风险驱动用例生成矩阵`
- `场景 x 边界/异常覆盖矩阵`
- `接口 x 字段语义等价类矩阵`
- `测试用例`
- `断言计划`
- `测试数据计划`
- `边界测试`
- `安全测试`
- `跨接口一致性测试`
- `接口编排/集成测试`

`结构化路由影响分析` 必须记录是否存在当前项目可用的结构化路由分析工具。可用时写明工具或来源、结构化输出位置和置信度；不可用时写明兜底方式，兜底方式必须来自代码 diff、路由定义、API 文档、代码搜索、项目文档、Agent 推理或人工补证。

`变更代码影响接口清单` 必须把结构化路由影响分析或兜底分析发现的变更文件/符号映射到直接影响接口、间接受影响接口、逻辑类似接口、调用方/消费者/任务和覆盖方式。每个已纳入或部分纳入测试范围的影响接口必须有覆盖方式；未纳入时必须有豁免依据。

每个核心业务场景必须关联至少一个用例或矩阵单元。每个必测场景覆盖单元和必测矩阵单元必须关联用例 ID。每个核心用例必须有关联断言计划和证据采集要求。

## Reviewer 契约

`测试用例评审.md` 必须包含：

- `业务场景拆解评审`
- `变更代码影响接口清单评审`
- `多角色评审结论矩阵`
- `角色冲突与裁决`
- `场景 x 边界/异常覆盖矩阵评审`
- `接口 x 字段语义等价类矩阵评审`
- `Anti-Happy-Path Gate`
- `最终评审结果`

多角色评审结论矩阵必须包含资深测试工程师、资深产品经理、资深软件工程师、安全工程师和测试证据审计员。任一角色存在未解决阻塞项时，最终结果不得为 `通过` 或 `有条件通过`。

触发 Reject If 时，最终结果只能为 `不通过` 或 `阻塞`。

## Executor 契约

`执行记录.md` 必须包含：

- `单接口请求记录`
- `断言执行记录`
- `证据包`
- `接口 x 字段语义等价类矩阵执行状态`
- `场景 x 边界/异常覆盖完整性` 或等价执行记录
- `跨接口一致性执行记录`
- `DB 验证记录`

执行状态只能为：

- `已执行`
- `跳过`
- `阻塞`

用例结果只能为：

- `通过`
- `失败`
- `阻塞`
- `跳过`

`已执行` 的必测单元必须有证据路径。每条通过断言必须有证据路径。写接口必须提供 DB 或副作用验证；跨接口一致性用例必须为每个参与接口提供独立请求证据。

## Reporter 契约

`测试报告.md` 必须包含：

- `测试执行汇总`
- `接口 x 字段语义等价类矩阵执行完整性`
- `场景 x 边界/异常覆盖完整性`
- `覆盖率评估`
- `覆盖与证据可信度评分`
- `最终总结`

`覆盖与证据可信度评分` 章节必须同时输出覆盖可信度、证据可信度、综合可信度、可信度等级和是否允许作为发布门禁依据。

报告结论为 `通过` 时，必须满足：

- 无未通过用例。
- 无阻塞用例。
- 无未执行计划用例。
- 所有必测矩阵单元已执行且有证据路径。
- 所有必测场景覆盖单元已执行且有证据路径。
- 所有写接口副作用已有 DB 验证。
- 跨接口一致性必测项已执行。
- 覆盖可信度和证据可信度满足门禁，综合可信度不低于 85。

任一条件不满足时，不得给出发布通过建议。
