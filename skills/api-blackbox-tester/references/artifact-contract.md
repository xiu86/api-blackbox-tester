# 测试 Artifact 契约

本契约用于约束 `测试方案.md`、`测试用例评审.md`、`执行记录.md` 和 `测试报告.md` 的可追踪关系。reporter 生成最终结论前，应使用 `scripts/validate_test_artifacts.py` 检查本契约。

## 必须存在的阶段文件

同一测试目录中应包含：

- `测试方案.md`
- `测试用例评审.md`
- `执行记录.md`
- `测试报告.md`

缺少前序阶段文件时，报告不得结论为 `通过`。

## ID 前缀建议

- 业务场景：`SCN-001`
- 场景覆盖单元：`SCENE-001`
- 普通用例：`API-001`
- 边界用例：`BND-001`
- 安全用例：`SEC-001`
- 矩阵单元：`MATRIX-001`
- 跨接口一致性：`CONS-001`
- 编排链路：`FLOW-001`
- DB 验证：`DB-001`
- 问题：`ISSUE-001`

## Planner 契约

`测试方案.md` 必须包含：

- `业务场景拆解`
- `场景 x 边界/异常覆盖矩阵`
- `接口 x 字段语义等价类矩阵`
- `测试用例`
- `边界测试`
- `安全测试`
- `跨接口一致性测试`
- `接口编排/集成测试`

每个核心业务场景必须关联至少一个用例或矩阵单元。每个必测场景覆盖单元和必测矩阵单元必须关联用例 ID。

## Reviewer 契约

`测试用例评审.md` 必须包含：

- `业务场景拆解评审`
- `场景 x 边界/异常覆盖矩阵评审`
- `接口 x 字段语义等价类矩阵评审`
- `Anti-Happy-Path Gate`
- `最终评审结果`

触发 Reject If 时，最终结果只能为 `不通过` 或 `阻塞`。

## Executor 契约

`执行记录.md` 必须包含：

- `单接口请求记录`
- `接口 x 字段语义等价类矩阵执行状态`
- `场景 x 边界/异常覆盖完整性` 或等价执行记录
- `跨接口一致性执行记录`
- `DB 验证记录`

执行状态只能为：

- `已执行`
- `跳过`
- `阻塞`

用例结果只能为：

- `通过`
- `失败`
- `阻塞`
- `跳过`

`已执行` 的必测单元必须有证据路径。

## Reporter 契约

`测试报告.md` 必须包含：

- `测试执行汇总`
- `接口 x 字段语义等价类矩阵执行完整性`
- `场景 x 边界/异常覆盖完整性`
- `覆盖率评估`
- `测试结果可信度评分`
- `最终总结`

报告结论为 `通过` 时，必须满足：

- 无未通过用例。
- 无阻塞用例。
- 无未执行计划用例。
- 所有必测矩阵单元已执行且有证据路径。
- 所有必测场景覆盖单元已执行且有证据路径。
- 所有写接口副作用已有 DB 验证。
- 跨接口一致性必测项已执行。
- 测试结果可信度评分不低于 85。

任一条件不满足时，不得给出发布通过建议。
