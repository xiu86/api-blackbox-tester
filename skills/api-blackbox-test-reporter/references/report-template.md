# 接口黑盒测试报告模板

执行接口测试后使用本结构输出报告。

## 概览

- 测试目标：方法、路径、环境、构建版本或提交。
- 最终结论：通过、失败、阻塞或部分通过。
- 执行范围：通过、失败、跳过、阻塞的用例数量。
- 矩阵单元执行：已执行、跳过、阻塞的必测矩阵单元数量。
- 兼容性结论：通过、失败、不适用或证据不足。
- 覆盖率结论：充分、基本充分、不足或阻塞。

## 编码前测试方案摘要

| 类别 | 摘要 | 备注 |
| --- | --- | --- |
| 测试范围 | 范围内/范围外 | 说明假设 |
| 影响范围初判 | 直接影响、潜在扩散影响、证据来源和置信度 | 说明公共方法、共享组件、关联接口或业务链路影响 |
| 兼容性验证 | 计划验证的契约点 | 说明风险 |
| 测试数据 | 计划准备的数据 | 说明构造或获取方式 |
| 测试用例 | 计划用例数量和重点场景 | 说明优先级 |
| 边界测试 | 计划边界参数和值 | 说明风险 |
| 安全测试 | 计划安全场景和 payload | 说明隔离要求 |
| 接口 x 字段语义等价类矩阵 | 计划矩阵单元数量、必测数量和关键字段 | 说明账号/userID/靓号/不存在/无手机号等关键等价类 |
| 跨接口一致性测试 | 逻辑类似接口组和一致性用例数量 | 说明哪些接口必须分别执行，不能互相替代 |
| 补充验证项 | 需求未列举但应验证的场景 | 说明来源 |
| 接口编排/集成测试 | 计划链路、步骤和依赖 | 说明是否包含非本次变更接口、数据传递和分支 |
| 性能测试建议 | 建议性能测试的接口或链路 | 说明原因、场景和指标 |

## 失败项

每个失败项说明：

- 用例 ID。
- 可复现请求。
- 实际响应和状态码。
- 预期行为。
- 数据库证据。
- 根因分析：基于响应、DB、日志、配置、数据、环境或需求契约证据；证据不足时说明还需要补充哪些定位证据。
- 解决建议：给出可执行修复动作。
- 责任阶段：开发修复、测试数据准备、环境/运维、需求澄清、测试方案补充或安全授权。
- 复验范围：需要重跑的用例 ID、接口、链路 ID、DB 校验和兼容性/安全回归项。

| 问题ID | 用例/链路ID | 问题类型 | 证据 | 根因分析 | 解决建议 | 责任阶段 | 复验范围 |

| ISSUE-001 | API-001 | 失败/部分通过/阻塞/风险 | 响应、DB 或环境证据 | 已确认或疑似根因 | 可执行修复动作 | 开发修复 | 重跑 API-001 和相关 DB 校验 |

## 兼容性说明

记录以下契约是否变化：

- HTTP 状态码。
- 响应结构和必需字段。
- 错误码和错误消息契约。
- 分页、排序、筛选、幂等和鉴权行为。
- 边界参数处理和非法输入响应。
- XSS、注入、越权、敏感信息泄露等安全风险防护。
- 多接口链路中的字段传递、状态流转、步骤顺序、非变更接口依赖和失败处理语义。
- 关键接口或链路的性能特征，例如响应时间、分页默认值、返回数据量、缓存命中路径或依赖调用次数。
- 现有客户端可感知的数据库副作用。
