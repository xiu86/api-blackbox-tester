# 接口黑盒测试报告模板

执行接口测试后使用本结构输出报告。

## 概览

- 测试目标：方法、路径、环境、构建版本或提交。
- 最终结论：通过、失败、阻塞或部分通过。
- 执行范围：计划、已执行、通过、未通过、跳过、阻塞、未执行的用例数量。
- 测试成果：已执行测试用例、发现 Bug、已修复 Bug、复验通过 Bug、待修复 Bug、阻塞 Bug、疑似 Bug、非 Bug 风险的数量。
- 未通过根因简述：按问题或根因类别汇总；无未通过时写“无”。
- 矩阵单元执行：已执行、跳过、阻塞的必测矩阵单元数量。
- 业务场景执行：已覆盖、未覆盖、阻塞的核心业务场景数量。
- 覆盖可信度评分：0-100；证据可信度评分：0-100；综合可信度评分：0-100；可信度等级：高、中、低。
- 兼容性结论：通过、失败、不适用或证据不足。
- 覆盖率结论：充分、基本充分、不足或阻塞。

## 兼容性说明

记录以下契约是否变化：

- HTTP 状态码。
- 响应结构和必需字段。

- 边界参数处理和非法输入响应。
